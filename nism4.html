    Technical Challenges Update

    The other thing that I spent time on this week was trying to solve my e-portfolio challenges, specifically the need to dedicate significant amounts of time just manually hand coding HTML/CSS tags. For instance, my workflow for the collaborative discussion posts is currently something like this:

    1. Write the post in Google Docs
    2. Copy the post into the advanced editor on the Essex Online platform and format it
    3. Copy the post from the EO platform and paste it into an html document in GitHub
    4. Manually hand code all of the header, paragraph, list, and other required tags

    This takes far more time than it should, and makes updating the e-portfolio a dreaded chore. I have tried using the GitHub HTML editor, but it really does not help at all when pasting text in. I've tried putting the text into OpenOffice and saving it as an HTML file, but that just adds a ton of extra CSS tags that I then manually have to fix.

    What I really need is either a WYSIWYG editor that just outputs plain HTML tags rather than CSS, or an editor where I can paste in some text, highlight it, and tell it to add opening and closing tags around it. It really shouldn't be that hard to find, and yet it is.
