Seminar 2: TCP/IP v ISO/OSI

Read: Russell, A.L. (2006) ‘Rough Consensus and Running Code’ and the Internet-OSI Standards War. IEEE Annals of the History of Computing. Available from: https://www2.cs.duke.edu/courses/common/compsci092/papers/govern/consensus.pdf

Then consider the following:

“Would the Internet we have today be much better if it was based on the ISO/ OSI 7-layer model rather than TCP/IP? “

The debate about OSI vs TCP/IP reminds me a lot about a similar, and also largely now forgotten, debate between GNU and Linux. Most people have now heard of Linux, but very few now remember the GNU operating system project. GNU was the first real attempt at developing an open source alternative to the UNIX operating system. GNU Hurd was to be the crowning achievement; the kernel for this new operating system.

The single biggest technical leap forward for Hurd was supposed to be the use of a brand new microkernel architecture (nixFAQ, 2021). In theory, a microkernel architecture would far surpass the capabilities of the monolithic kernel architectures that were in use at the time. In practice, it turned out that actually creating a microkernel was far harder than expected, and the GNU Project struggled for years to develop a working version. When Linus Torvalds released the Linux kernel in 1992, the GNU Project scoffed at it as another monolithic relic. Of course we all know what happened; Linux exploded in popularity, while GNU Hurd languished. To this day, there are no microkernel operating systems in widespread use.

The lesson of GNU Hurd vs Linux is the same as with OSI vs TCP/IP; theoretical advantages of one architecture over another are entirely negated by practicality and time-to-market. So yes, OSI has many theoretical advantages over TCP/IP (Russell, 2006), and perhaps an Internet based on OSI would be more secure and more efficent. However if we had to wait for a fully operational OSI networking stack, we might still be waiting, just as we are still waiting for a working Hurd kernel.

Technical Challenges Update

The other thing that I spent time on this week was trying to solve my e-portfolio challenges, specifically the need to dedicate significant amounts of time just manually hand coding HTML/CSS tags. For instance, my workflow for the collaborative discussion posts is currently something like this:

  1. Write the post in Google Docs
  2. Copy the post into the advanced editor on the Essex Online platform and format it
  3. Copy the post from the EO platform and paste it into an html document in GitHub
  4. Manually hand code all of the header, paragraph, list, and other required tags

This takes far more time than it should, and makes updating the e-portfolio a dreaded chore. I have tried using the GitHub HTML editor, but it really does not help at all when pasting text in. I've tried putting the text into OpenOffice and saving it as an HTML file, but that just adds a ton of extra CSS tags that I then manually have to fix.

What I really need is either a WYSIWYG editor that just outputs plain HTML tags rather than CSS, or an editor where I can paste in some text, highlight it, and tell it to add opening and closing tags around it. It really shouldn't be that hard to find, and yet it is.
