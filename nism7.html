The articles give suggestions on different ways Kali Linux can be used too. For example, you could download a ready to use Debian based Kali Linux and set it up as a virtual machine, or you could create a Debian based virtual machine and install Kali Linux tools on it as you need it.

You can also get a "Live" distro. Essentially you make a bootable USB drive with Kali (or other distro) on it. Your computer can then boot using the USB drive instead of your hard drive, and will load up the OS on the USB drive.

I think the main issue for me would be my lack of knowledge of Linux OS and learning the commands you must use for each security tool. Also, learning to understand what each scan result means.

It is possible to bring down the target system. In my very first job as a security analyst for a small bank, I ran a simple port scan on our backup banking system, which was this big HP-UX server. Unbeknownst to anyone, HP-UX had a denial of service vulnerability at the time where connections to a specific port number caused the entire OS to become unresponsive. It was a really good thing that I was testing the backup server rather than production.

The other big potential issue is getting banned by your ISP for suspected hacking.

I think the criteria such as Powerful, is very subjective. For example, the SQL map may not be a very versatile tool, but it does a great job at automatic SQL injection testing, and so, when it comes to these types of attacks, should it not be rather flagged as powerful?

I don't mind the criteria, but the rating system of binary Yes or No is too simple imo.
